Introduction

Developed a Web app with Oauth2 based REST API of Codechef, a competitive programming website. The web app works as a recommendation engine that consumes the REST API and recommends new questions to the user according to their previously solved problems on the platform.

Recommendation of new questions is done with the help of a K-means classifier which works as a tag-based recommendation model. The web app also uses an asynchronous task queue to update all the required user tokens and recommendation list for an user.

Main features of the application –

  • A custom authentication system which works with CodeChef Oauth2 REST API
  • Scrape questions alongside tag and difficulty in bulk using async requests
  • User can get new problems to solve based on their previously solved problems and user ranking
  • User can add recommended problems to their Todo list
  • User tokens and solved problems are updated asynchronously without user intervention
  • K-means model specifically trained on problems and tags from the codechef website

Tech Stack

The project can be broadly categorized into two sections

1. Data pipelining and model training

This part of the application utilized REST API to scrape problems alongside there tags from the CodeChef platform. The scraping is done with the help of asynchronous requests to maximize the throughput of the application.

These scraped problems than can be used to train a K-means model after it is pre-processed accordingly. Handling special characters, null value removal, tokenization, etc.

  • After data preprocessing scikit-learn was used to train an ML model
  • And finally, the ML model was stored into a persistent format for later use

2. Web application

  • The web application is written in Django and uses MySQL as the main database
  • To create and update the recommendation list celery is used, as an asynchronous task is required
  • Redis server is also used as a broker for handling async task with celery
  • Supervisord is used to automate the task of running and handling async part of the code
  • Once user logs in and application have the list of all previously solved problems by the user, it then pings the ML model to get tags for the next set of problems that should be ideal for user to solve based on the user solution history.
